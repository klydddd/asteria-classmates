BosesPH Toolkit

A reusable open-source pipeline that turns raw Philippine-language speech recordings and transcripts into clean datasets, ASR benchmarks, WER/CER reports, and optional fine-tuned Whisper models.

End-to-end sequence

Pipeline overview

The workflow is intentionally ordered so each artifact can be reproduced from the previous one and reviewed before it becomes training or benchmark data. For a full vertical diagram, open the pipeline map.

1

Import

Parse a PLD session, match WAV files to transcripts, inspect audio, and standardize files.

2

Normalize

Clean Unicode, whitespace, punctuation spacing, repeated punctuation, and transcript tags.

3

Review

Approve, fix, reject, skip, and resume clip decisions from the terminal.

4

Build

Filter approved clips and create speaker-aware train, validation, and test splits.

5

Transcribe

Run baseline or fine-tuned ASR on one file or an entire dataset split.

6

Evaluate

Compute WER and CER with normalized scoring text and write benchmark reports.

7

Fine-tune

Optionally train Whisper with LoRA and compare it against the baseline model.

Review is a quality gate: only approved clips are included when building train, validation, and test splits.

What the toolkit does

Capabilities

Each capability is available from the core Python services and is exposed through one or more interfaces for developers, reviewers, dashboards, and agents.

Import PLD sessions

Reads one recording session, parses metadata and transcripts, and produces a deterministic dataset folder.

Validate audio

Checks WAV structure, duration, sample rate, channels, sample width, corruption, silence, and quiet clips.

Standardize audio

Copies or converts readable clips to mono, 16 kHz, signed 16-bit PCM WAV for ASR training.

Normalize transcripts

Applies NFC Unicode, whitespace, punctuation, and allowed tag normalization in metadata.

Review clips

Provides a resumable terminal workflow for approving, rejecting, or flagging clips that need fixes.

Build datasets

Generates speaker-aware train, validation, and test CSVs with reproducible split seeds.

Transcribe audio

Runs Hugging Face ASR models on a single clip or a split and writes predictions CSV files.

Evaluate metrics

Computes word error rate and character error rate with text normalized for scoring.

Fine-tune Whisper

Uses Hugging Face Seq2SeqTrainer with LoRA defaults, optional fp16, and memory-aware settings.

Compare models

Produces a Markdown comparison between baseline and fine-tuned benchmark results.

Export notebooks

Generates Colab notebooks for GPU fine-tuning and evaluation workflows.

Generate documentation

Writes dataset cards, model cards, JSON reports, benchmark reports, and metadata schemas.

CLI reference

Command reference

The toolkit is CLI-first. These are the commands a developer can run locally, in scripts, or inside notebooks and cloud jobs.

Install and verify

setup 
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[asr,train,api,mcp,dev]"

bosesph validate-metadata sample_data/metadata_template.csv
bosesph export-metadata-schema --output docs/metadata.schema.json
ruff check .
black --check .
pytest

Import, normalize, and review

data prep 
bosesph import-pld PLD/PAM/0400 --output outputs/dataset --overwrite
bosesph normalize-transcripts outputs/dataset
bosesph review outputs/dataset

Build reproducible splits

dataset 
bosesph build-dataset outputs/dataset \
  --output outputs/dataset_15spk \
  --train 0.70 --val 0.15 --test 0.15 --seed 42

Transcribe one file or a full split

asr 
bosesph transcribe outputs/dataset_15spk/audio/pam_000001.wav \
  --model openai/whisper-small --language tl

bosesph transcribe outputs/dataset_15spk \
  --model openai/whisper-small --language tl \
  --split test --output outputs/benchmark/baseline_predictions.csv \
  --limit 300

Evaluate WER and CER

benchmark 
bosesph evaluate \
  --predictions outputs/benchmark/baseline_predictions.csv \
  --output outputs/benchmark/baseline \
  --model-name "Baseline Whisper Small" \
  --language kapampangan

Fine-tune with LoRA

training 
bosesph finetune outputs/dataset_15spk \
  --output outputs/model/bosesph-kapampangan-v1 \
  --base-model openai/whisper-small \
  --language tl \
  --epochs 3 --batch-size 8 --learning-rate 1e-5 \
  --lora-r 32 --lora-alpha 64 --lora-dropout 0.05

Evaluate and compare the fine-tuned model

comparison 
bosesph transcribe outputs/dataset_15spk \
  --model outputs/model/bosesph-kapampangan-v1/model --language tl \
  --split test --output outputs/benchmark/finetuned_predictions.csv

bosesph evaluate \
  --predictions outputs/benchmark/finetuned_predictions.csv \
  --output outputs/benchmark/finetuned \
  --model-name "BosesPH LoRA Fine-tuned" \
  --language kapampangan

bosesph compare \
  --baseline outputs/benchmark/baseline/results.json \
  --finetuned outputs/benchmark/finetuned/results.json \
  --output outputs/benchmark/comparison.md

Export Colab notebooks

notebooks 
bosesph export-colab outputs/dataset_15spk \
  --output colab_finetune.ipynb \
  --base-model openai/whisper-small \
  --language tl --max-steps 500

bosesph export-eval-colab \
  --dataset-drive-path /content/drive/MyDrive/bosesph/dataset_15spk \
  --model-drive-path /content/drive/MyDrive/bosesph/model \
  --base-model openai/whisper-small \
  --language tl --eval-language kapampangan \
  --output colab_eval.ipynb

Quality and safety

Data standards and safety

Speech data needs strict format rules, explicit review states, and privacy protections before it becomes reusable benchmark data.

Required metadata

  • audio_id, file_path, transcript, language
  • speaker_id, duration_seconds, sample_rate
  • split, quality_status, reviewer notes and provenance fields

Transcript tags

  • [noise] for overlapping non-speech sound
  • [laughter] for audible laughter
  • [unclear] for speech that cannot be transcribed reliably
  • [silence] for meaningful silence within a clip

Language codes

  • pam Kapampangan pilot language
  • tgl, ceb, ilo, hil and other ISO 639-3 codes
  • eng;fil style code-switch metadata when needed

Deterministic IDs

Audio IDs use language prefixes and six digits, such as pam_000001. Speaker IDs stay anonymous, such as spk_001.

Review statuses

Clips move through pending, needs review, approved, and rejected states. Only approved clips enter dataset builds.

Repository safety

Do not commit secrets, .env files, private audio, personal speaker metadata, generated outputs/, or model weights.

Local walkthrough

Run it locally

Start with the CLI, inspect outputs, then launch the optional API and web dashboard for a visual demo.

1. Prepare

Create a Python 3.10+ environment, install the package with extras, and validate the sample metadata schema.

2. Build data

Import PLD audio, normalize transcripts, complete review, and build deterministic train/validation/test splits.

3. Benchmark

Run Whisper baseline transcription, evaluate WER/CER, optionally fine-tune with LoRA, then compare models.
